# Phase 3a — Generate TypeScript Types from Go

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate the silent drift between the Go backend types and the hand-written `web/lib/types.ts` by generating the TypeScript response/domain types directly from the Go structs, with a CI check that fails when they're out of date.

**Architecture:** Use [`tygo`](https://github.com/gzuidhof/tygo) to transpile the Go structs in `internal/domain` and the response structs in `internal/service` into a generated `web/lib/types.gen.ts`. A thin hand-written `web/lib/types.ts` re-exports the generated types and adds the few things codegen can't model (string-literal enum unions and request DTOs). A `gen:types` script regenerates; a CI/local drift check runs it and fails if the working tree changed.

**Tech Stack:** Go 1.26 + tygo (Go module tool) → TypeScript; Next.js 14 `web/`.

**Design decisions (flag before executing):**
- **Tool: `tygo`** — purpose-built Go→TS struct transpiler, respects `json` tags, maps `time.Time`→`string` and `int64`→`number`. Lighter than a full OpenAPI pipeline, and the backend has no OpenAPI spec today.
- **Scope: response + domain types only.** Request bodies are inline anonymous structs in handlers (e.g. `createExpense(pid, b: any)`); they don't codegen. They stay hand-typed as small DTOs in `types.ts`. This still removes the large majority of drift risk (every entity + every GET response).
- **Enums stay hand-written.** tygo renders typed string consts as `type X = string`; we keep the precise unions (`"expense" | "savings" | "pay"`) by hand in `types.ts` — they change rarely and the precision is worth it.
- **Generated file is committed** (not git-ignored) so the frontend builds without a Go toolchain and the drift check is a simple `git diff`.

- [ ] **Step 1: Install tygo as a module tool**

Run:
```bash
cd CostCalculator/backend && go get github.com/gzuidhof/tygo@latest
```
Expected: `go.mod`/`go.sum` updated with the tygo dependency.

- [ ] **Step 2: Write the tygo config**

Create `CostCalculator/backend/tools/tygo.yaml`:

```yaml
packages:
  - path: "costcalculator/backend/internal/domain"
    output_path: "../web/lib/types.gen.ts"
    type_mappings:
      time.Time: "string"
      int64: "number"
    frontmatter: |
      // Code generated by tygo from internal/domain + internal/service. DO NOT EDIT.
      // Regenerate with: backend/tools/gen-types.sh
  - path: "costcalculator/backend/internal/service"
    output_path: "../web/lib/types.gen.ts"
    type_mappings:
      time.Time: "string"
      int64: "number"
    include_files:
      - "summary.go"
```

Notes: the second package entry appends the `service` response structs (`PeriodSummary`, `PeriodTrends`, `AccountStatus`, `DailySpend`, `CategoryTotal`, `TrendPoint`, `CategoryComparison`, `SavingsHistoryPoint`, `LendTotals`, `WindowWithStatus`) from `summary.go` to the same output file.

- [ ] **Step 3: Write the generation wrapper**

Create `CostCalculator/backend/tools/gen-types.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
go run github.com/gzuidhof/tygo generate --config tools/tygo.yaml
echo "generated web/lib/types.gen.ts"
```

Make it executable:
```bash
chmod +x CostCalculator/backend/tools/gen-types.sh
```

- [ ] **Step 4: Generate the types and inspect the output**

Run:

```bash
cd CostCalculator/backend && ./tools/gen-types.sh
```
Expected: `web/lib/types.gen.ts` created. Open it and confirm it contains `Expense`, `Period`, `Account`, `Budget`, `PeriodSummary`, `PeriodTrends`, etc., with `number` for amounts and `string` for dates. If tygo emits a Go enum as `type CategoryKind = string`, that's expected — the precise union is added in Task 2.

### Task 2: Make `types.ts` re-export the generated types

**Files:**
- Modify: `CostCalculator/web/lib/types.ts`

- [ ] **Step 1: Replace generated-equivalent declarations with re-exports**

Rewrite `CostCalculator/web/lib/types.ts` so it (a) re-exports everything generated, (b) keeps the precise enum unions, and (c) keeps `AuthResult`/`TokenPair` and request DTOs that aren't generated:

```ts
// Hand-maintained edge of the type layer. Entity + response shapes come from
// the Go backend via codegen (types.gen.ts). This file adds the precise enum
// unions tygo can't infer, plus auth/request DTOs that have no Go struct.
export * from "./types.gen";

// Precise string-literal unions (Go renders these as `string`).
export type CategoryKind = "expense" | "savings" | "pay";
export type AccountKind = "bank" | "mobile" | "cash" | "savings" | "virtual";
export type VirtualRole = "external" | "lendGiven" | "lendTaken";
export type PeriodStatus = "open" | "closed";
export type LendType = "given" | "taken";
export type LendStatus = "open" | "settled";
export type WindowState = "upcoming" | "active" | "expired" | "paid";

// Auth payloads (no single Go struct backs these response envelopes).
export interface TokenPair { accessToken: string; refreshToken: string; }
export interface AuthResult { user: import("./types.gen").User; tokens: TokenPair; }

// Import report (lives in internal/importer, not codegen'd here).
export interface ImportSheetReport { sheet: string; skipped: boolean; expenses: number; transfers: number; budgetItems: number; lends: number; warnings?: string[]; }
export interface ImportReport { sheets: ImportSheetReport[]; }
```

> If `types.gen.ts` already exports `CategoryKind`/`AccountKind` etc. as `string` aliases, tygo's export and the union above will collide. Resolve by adding those enum type names to a tygo `exclude` list in `tools/tygo.yaml` (or post-process to drop them) so the precise unions here win. Re-run `gen-types.sh` after adjusting and confirm no duplicate-export TS error in Step 2.

### Task 3: Drift check

**Files:**
- Create: `CostCalculator/backend/tools/check-types.sh`

- [ ] **Step 1: Write the drift check**

Create `CostCalculator/backend/tools/check-types.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
./tools/gen-types.sh
if ! git diff --quiet -- ../web/lib/types.gen.ts; then
  echo "ERROR: web/lib/types.gen.ts is out of date. Run backend/tools/gen-types.sh and commit." >&2
  git --no-pager diff -- ../web/lib/types.gen.ts
  exit 1
fi
echo "types.gen.ts is up to date"
```

Make it executable:
```bash
chmod +x CostCalculator/backend/tools/check-types.sh
```

- [ ] **Step 2: Verify the check passes on a clean tree**

Run: `cd CostCalculator/backend && ./tools/check-types.sh`
Expected: prints `types.gen.ts is up to date`, exit 0.

- [ ] **Step 3: Verify the check fails on drift (manual sanity)**

Temporarily add a field to a Go struct (e.g. add `TestField string \`json:"testField"\`` to `domain.Expense`), run `./tools/check-types.sh`, confirm it exits non-zero and shows the diff, then revert the Go change and re-run to confirm green.

- [ ] **Step 4: Commit**

```bash
git add CostCalculator/backend/tools/check-types.sh
git commit -m "ci(types): drift check for generated TypeScript types"
```
